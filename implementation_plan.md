# 1943 KAI Game Completion Plan

이 프로젝트는 오락실 명작 게임인 '1943'의 느낌을 저작권에 저촉되지 않는 선(절차적 그래픽 생성, 'Pacific Storm 1943' 타이틀 등)에서 최대한 비슷하게 구현하기 위해 설계된 Pygame 기반 프로젝트입니다. 

대부분의 핵심 컴포넌트(플레이어 전투기, 다양한 적 편대와 AI, 스테이지 매니저, 화면 스크롤, 사운드 관리자, 폭발 이펙트, HUD UI 등)가 개발되어 있으나, 이를 유기적으로 엮어 게임을 실행하는 메인 루프(`game.py`)가 아직 완성되지 않았고 일부 설정이 누락되어 있습니다.

이에 게임이 원활히 동작하도록 통합하고 게임 오버/클리어 흐름을 구현하는 작업을 수행하고자 합니다.

### [Main Game Logic]

#### [MODIFY] [game.py](file:///c:/Users/kmw84/PycharmProjects/1943_KAI/game.py)
- `game.py` 파일에 다음과 같은 핵심 통합 기능을 구현합니다.
  - **초기화 및 로드**:
    - 최고 점수 로딩 (`save/highscore.json` 연동)
    - 플레이어, 사운드 매니저, BGM 매니저, 스테이지 매니저, 파티클 이펙트, UI 스크린 인스턴스화
  - **게임 상태 관리 (Game State Machine)**:
    - `TITLE` (타이틀 화면)
    - `PLAYING` (게임 진행 중)
    - `PAUSED` (일시 정지)
    - `STAGE_CLEAR` (스테이지 클리어 컷신 및 정산)
    - `GAME_OVER` (게임 오버 화면)
    - `GAME_CLEAR` (최종 엔딩 화면)
  - **이벤트 핸들러**:
    - 각 상태에 따른 키 입력 분기 처리
    - `PLAYING` 상태에서 폭탄 사용(`X` 키) 시 화면에 존재하는 모든 적군 탄환 제거 및 적군 대미지(메가크래시) 및 플레이어 에너지 소모 구현
  - **업데이트 루프**:
    - `bullet.collision.check_all`을 사용한 탄환, 적, 플레이어, 아이템 간 충돌 검사 및 폭발 이펙트 트리거
    - 플레이어의 에너지 소모 및 사망(라이프 감소 및 게임오버) 조건 체크
    - 스테이지 보스 처치 여부에 따른 스테이지 클리어 연출 및 다음 스테이지 이동
  - **렌더링 루프**:
    - 스테이지 배경 스크롤, 플레이어/적/탄환/아이템/이펙트 스프라이트 그리기
    - 게임 UI(에너지 게이지, 현재 스코어, 최고 점수)를 화면에 오버레이
    - 일시정지, 게임오버, 타이틀 화면을 상태에 따라 렌더링

## Verification Plan

### Automated Tests
- Pygame 환경이 잘 기동되는지 `main.py` 실행을 제안하고 직접 동작 검증을 하도록 하겠습니다.
  `python main.py`

### Manual Verification
- **타이틀 화면**: 스페이스바 혹은 엔터 키로 게임이 시작되는지 확인.
- **게임 플레이**: 방향키/WASD로 비행기가 움직이고 `Z` 키로 투사체가 발사되는지 확인.
- **충돌 및 대미지**: 적기 충돌 혹은 피탄 시 에너지가 깎이고, 에너지가 0이 되면 라이프가 감소하는지 확인.
- **아이템 습득**: 아이템(에너지 충전, 무기 강화 등) 획득 시 플레이어 스펙이 변하는지 검사.
- **폭탄 사용**: `X` 키를 눌렀을 때 전체 화면 플래시와 함께 탄환이 소멸하고 적이 데미지를 입는지 확인.
- **스테이지 클리어**: 스테이지 1 보스를 물리쳤을 때 스테이지 클리어 판정이 나고 다음 스테이지로 넘어가는지 확인.
- **최고 점수 저장**: 게임 오버 시 획득한 최고 점수가 보존되고 타이틀/게임오버 화면에 나타나는지 확인.
